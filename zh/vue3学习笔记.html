# 认识vue3

vue3模板中可以不需要根标签

# setup

# setup细节

# 执行时机

  • setup在生命周期beforeCreate之前就执行了，只执行一次
  • 可推断出，setup执行的时候，组件还没有创建出来，组件实例对象this不能用
  • this是undefined，那么data/computed/methodspros也不能调用

# setup返回值

  • 返回值是一个对象，内部属性方法是给html模板使用的
  • vue3中尽量不要混合使用，优先setup中的
  • 参数props：包含props配置声明且传入了所有属性的对象
  • context参数：是一个对象，里面有attrs对象（获取当前组件标签上的所有的属性的对象，但是该属性是在props中没有声明接收的所有声明对象），相当于this.$attrs，emit方法，slot对象

# ref

  • 组合API中第一个要使用的函数
  • 只执行一次
  • ref是一个函数：定义一个响应式数据，返回一个ref对象，有一个value属性需要对数据进行操作的话，需要操作其value值；而在html中不需要使用value值
  • 用来定义基本类型的响应式数据
  • count是一个ref对象
  • 如果将ref里传入对象，就会变成Proxy类型对象
  • ref另一个作用是获取网页元素标签
  setup(){
    // let count = 1  // 不是响应式数据
    // 变量
    const count = ref(0)
    // 方法
    function add(){
      // count是一个ref对象。不能进行++操作
      
      count.value++
    }
    return {
      count,
      add
    }
  }

# reactive

  • 定义多个响应式数据

  • 返回的是一个Proxy代理对象，被代理的目标对象是obj对象

  • user是代理对象，obj是目标对象

  • const user = reactive(obj)

  • 直接使用目标obj对象方式更新数据不是响应式的，只能使用代理

  • 如果操作代理对象，目标对象中数据也会随之变化，同时如果想要操作数据的时候，界面动态渲染，那么也是操作代理对象

# vue3的响应式

  • Proxy代理

new Proxy(参数1，参数2)

参数1：user--》target目标对象

参数2handler：--》处理器对象，用来监视数据

const userProxy = new Proxy(user,{
  get(target,props){
    console.log('get方法调用了',target,props);
    return Reflect.get(target,props) // 目标对象，操作的key键
  },
  // 可以动态修改添加目标对象的属性
  set(target,props,val){
    console.log('set方法调用了');
    return Reflect.set(target,props,val) // 目标对象，操作的key键，传入的value值
  },
  deleteProperty(target,props){
    console.log('删除了');
    return Reflect.deleteProperty(target,props)
  }
})

  • reflect反射

# 计算属性与监视

# 计算属性

  • computed
  • 如果计算属性只传入一个回调函数，表示的是`get
  • 返回的是一个ref对象

# 监视

  • watch 监视的数据变化时会触发，加载完成不会触发

  • watchEffect 会立即执行。不需要配置immediate:true

  • watch(user,({firstName, lastName})=>{ // 监视user
        fullName3.value = firstName+'_'+lastName
      },{immediate:true,deep:true}) //立马执行。深度监视

  • 监视非响应式数据的时候，需要用回调才能监视到，如果直接写 user.firstName 无法监视到

watch([()=>user.firstName,()=>user.lastName],()=>{
        console.log('1112');
      })

# toRefs

  • 如果在 return 中返回一个 reactive 解构赋值的值，那么这个值不是响应式的
  • 用 toRefs 在 return 时返回的每个对象都会变成响应式的ref
  • 或者返回return {...toRefs(state)}
